# 06 - 技能系统：Claude 的武功秘籍

## 一句话总结

技能就是预录好的"操作指南"，告诉 Claude 遇到特定任务时按什么步骤来。就像武功秘籍，照着练就行。

---

## 通俗理解

工具是"招式"（读文件、写代码、跑命令），技能是"套路"（把多个招式组合成一套完整的打法）。

```
工具 (招式):
  Read, Write, Edit, Bash, Grep, Glob ...

技能 (套路):
  /batch  → 大规模并行变更的套路
  /loop   → 定时重复任务的套路
  /verify → 验证代码变更的套路
  /debug  → 调试问题的套路
  /stuck  → 卡住时的恢复套路
```

**类比**：
- 工具 = 菜刀、锅、铲子
- 技能 = 红烧肉的菜谱（第一步热油，第二步放肉，第三步加酱油...）

---

## /batch：最强大的技能

这是 Claude Code 最令人印象深刻的技能。用一句话概括：

**把一个大任务拆成 20 个小任务，然后召唤 20 个分身同时干。**

```
你: /batch 把所有 React 组件迁移到 Vue

Claude 的执行流程:

Phase 1: 研究和规划 (先想清楚)
  ├── 召唤一个分身深入研究代码库
  ├── 把任务拆成 20 个独立的工作单元
  │   ├── 单元 1: 迁移 src/components/
  │   ├── 单元 2: 迁移 src/hooks/
  │   ├── 单元 3: 迁移 src/utils/
  │   └── ... (共 20 个)
  ├── 确定测试方案
  └── 写计划 → 你审批

Phase 2: 并行执行 (开干!)
  ├── 召唤 20 个分身，每个在独立的 Git Worktree 里干活
  ├── 每个分身的流程:
  │   1. 改代码
  │   2. 简化代码
  │   3. 跑测试
  │   4. 创建 PR
  │   5. 报告: "PR: https://github.com/.../pull/123"
  └── 20 个分身同时工作！

Phase 3: 跟踪进度
  ├── # | 单元 | 状态 | PR
  ├── 1 | components/ | 完成 | github.com/.../12
  ├── 2 | hooks/     | 完成 | github.com/.../34
  ├── 3 | utils/     | 失败 | 测试不通过
  └── ...
```

## /loop：定时任务

```
你: /loop 5m 检查一下部署状态

Claude: 好的，每 5 分钟检查一次

  → 立即执行一次
  → 然后每 5 分钟自动执行
  → 直到你取消
```

就像你设了个闹钟，每隔一段时间提醒你做一件事。

## 技能注册：怎么添加新技能

```python
def register_skill(name, description, get_prompt):
    """
    name: 技能名称
    description: 描述
    get_prompt: 返回给 Claude 的操作指南
    """
    skills[name] = {
        "description": description,
        "get_prompt": get_prompt,
    }
```

技能本质上就是一段**精心编写的 prompt**，告诉 Claude 遇到这类任务时按什么步骤来。

---

## 附带文件：技能的"装备包"

有些技能需要额外的参考文件。比如 /verify 可能需要一个测试检查清单。

```
技能注册时声明:
  files = {
    "checklist.yaml": "1. 检查类型... 2. 检查边界...",
    "template.md": "# 测试报告模板...",
  }

首次使用时:
  → 把这些文件写到 ~/.claude/skills/verify-xxx/
  → 在 prompt 中告诉 Claude: "参考文件在 xxx 目录"

安全措施:
  ├── 随机目录名（防止被提前创建恶意文件）
  ├── 严格的权限控制（只有 owner 能访问）
  ├── 路径遍历检查（防止逃逸到其他目录）
  └── 失败不影响技能运行（优雅降级）
```

## 技能发现：97% 的查找都是白费

一个有趣的发现：97% 的技能查找什么都没找到。

旧方案：每次循环都同步查找技能 → 白白浪费时间
新方案：在 Claude 思考和执行工具的间隙异步查找 → 零额外延迟

```
旧方案:
  Claude思考(2s) → 查找技能(0.1s) → 执行工具(1s) → 查找技能(0.1s) → ...
  总延迟: 每轮多 0.1s

新方案:
  Claude思考(2s) ─┬─ 查找技能(0.1s) ← 同时进行!
  执行工具(1s) ────┘
  总延迟: 零额外延迟!
```

## 面试怎么答

**Q: 技能和工具的本质区别？**
A: 工具是原子操作（读文件、执行命令），技能是工作流模板（编排多个工具的使用方式）。技能本质上是一段精心设计的 prompt，告诉 Claude 如何完成特定类型的任务。

**Q: 为什么 /batch 需要 Git Worktree？**
A: /batch 启动 20 个分身并行改代码。如果都在同一个目录，20 个人同时编辑同一个文件就乱套了。Worktree 给每个人一个独立的代码副本，互不干扰。
