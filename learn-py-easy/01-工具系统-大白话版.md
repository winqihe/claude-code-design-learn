# 01 - 工具系统：给 AI 一双手

## 一句话总结

工具系统就是给 Claude 装上"手"和"眼睛"——让它能读文件、写代码、跑命令、搜网页。每个工具都像一把瑞士军刀，有统一的接口，但各自干不同的事。

---

## 通俗理解

想象你是一个超级聪明的盲人（Claude），你什么都知道，但你没法动手做事。

这时候有人给你递了 40 多把工具：

```
🔧 Read     → 能读文件内容
🔧 Write    → 能创建新文件
🔧 Edit     → 能修改已有文件
🔧 Bash     → 能执行终端命令（最强大的工具）
🔧 Grep     → 能在代码里搜索关键词
🔧 Glob     → 能按名字找文件（比如找所有 .py 文件）
🔧 WebFetch → 能访问网页
🔧 AgentTool→ 能召唤"分身"帮你干活
🔧 MCPTool  → 能连接外部工具（数据库、API 等）
```

Claude 自己决定用哪把工具、什么时候用。你的问题是"帮我修这个 bug"，Claude 会自己判断：先 Grep 搜相关代码 → Read 读文件 → Edit 改代码 → Bash 跑测试。

---

## 工具长什么样？

用 Python 来说，一个工具就是一个对象，必须回答这几个问题：

```python
class Tool:
    name: str           # 你叫什么？ "Bash"
    description: str    # 你能干嘛？ "执行终端命令"

    async def call(self, args, context):
        """真正干活的地方"""
        result = subprocess.run(args["command"], shell=True)
        return result.stdout

    is_concurrency_safe: bool  # 你能和别人一起干活吗？
    is_read_only: bool          # 你只看不摸吗？
```

就这么简单。但 Claude Code 的精妙之处在于**默认值设计**。

---

## 核心设计：安全第一

```python
TOOL_DEFAULTS = {
    "is_concurrency_safe": False,   # 默认：不能并行（安全）
    "is_read_only":         False,   # 默认：可能会改东西（安全）
    "is_destructive":       False,   # 默认：不会搞破坏（安全）
}
```

这就像公司制度：

- 新员工默认**不能并行处理多个任务**（怕出错）
- 新员工默认**可能有写权限**（宁可多检查，也不能漏查）
- 你必须**主动声明**"我这个工具是只读的、安全的"，才能被并行执行

这叫 **Fail-Closed**（宁可错杀，不可放过）。

## 工厂函数：造工具的流水线

```python
def build_tool(name, call, description, **kwargs):
    """
    你只需要告诉我：
    1. 工具叫什么
    2. 怎么干活
    3. 干什么用的

    安全相关的默认值我帮你填好
    """
    return Tool(
        name=name,
        call=call,
        description=description,
        is_concurrency_safe=False,  # 默认不安全
        is_read_only=False,          # 默认可写
        **kwargs,                    # 你可以覆盖默认值
    )
```

**类比**：这就像买车。你选了车型（call 函数），4S 店默认给你装了最安全的配置（安全气囊、ABS）。你想要运动模式？自己显式选配（覆盖默认值）。

## 进度报告：让用户看到 AI 在干嘛

工具执行可能要几秒钟，用户盯着空白屏幕会慌。所以每个工具都能实时报告进度：

```python
async def call(self, args, context, on_progress):
    on_progress("正在搜索...")
    results = await search(args.query)
    on_progress(f"找到 {len(results)} 个结果")
    return results
```

终端上你会看到：

```
⠋ 正在搜索...
✓ 找到 42 个结果
```

## 延迟加载：省钱的艺术

Claude 有 40 多个工具，但每次对话不一定全用到。如果把所有工具的说明都塞进 prompt，token 费用会很高。

解决方案：**按需加载**

```
常用工具（Read、Bash、Grep）→ 每次都告诉 Claude
冷门工具（MCP、AgentTool）   → Claude 需要时再去"翻说明书"
```

就像你办公桌上只放常用的笔和本子，不常用的工具放在抽屉里，需要时再拿出来。

## 面试怎么答

**Q: 为什么 is_concurrency_safe 默认 False？**
A: 安全第一。如果默认 True，一个有副作用的工具被并行执行可能导致数据损坏。宁可串行慢一点，也不能冒数据丢失的风险。

**Q: 工具系统最核心的设计思想是什么？**
A: 统一接口 + 安全默认值。所有工具实现相同的接口，Claude 不需要知道每个工具的细节。安全默认值确保新工具不会引入安全问题。
