# 02 - 运行循环：Claude 的心跳

## 一句话总结

运行循环就是 Claude 的"思考→行动→再思考"的无限循环，像心跳一样驱动着整个 Agent 系统。

---

## 通俗理解

你用过 ChatGPT 对吧？你问一个问题，它回答一次，结束。

Claude Code 不一样。它更像一个**不停工作的员工**：

```
你: "帮我修这个 bug"

Claude 的内心活动:
  1. 🤔 让我先搜一下相关代码...    (调用 Grep 工具)
  2. 📖 看看这个文件...            (调用 Read 工具)
  3. ✏️ 我觉得应该改这里...         (调用 Edit 工具)
  4. 🧪 让我跑一下测试...          (调用 Bash 工具)
  5. 😊 测试通过了！               (回答用户)
```

这个"思考→用工具→看结果→再思考"的过程，就是**运行循环（Query Loop）**。

---

## 代码长什么样？

```python
async def query_loop(user_message):
    messages = [user_message]

    while True:  # ← 无限循环，直到任务完成
        # 第一步：让 Claude 思考
        response = await call_claude_api(messages)

        if response.is_done:
            # Claude 说"我做完了"，退出循环
            return response

        if response.wants_to_use_tool:
            # Claude 说"我要用某个工具"，执行它
            tool_result = await execute_tool(response.tool_call)
            messages.append(tool_result)
            # 然后继续循环，让 Claude 看到结果再思考
```

就这么简单。但实际代码有 1700 多行，因为要处理无数边界情况。

## 上下文管理：防止 Claude "失忆"

Claude 的记忆有限（200K tokens）。对话太长，旧的消息会被挤出去。

Claude Code 有 4 层压缩策略，像整理房间一样从轻到重：

```
第一层：Snip（裁剪）
  像扔掉过期的报纸，删掉不再需要的中间消息

第二层：Microcompact（微压缩）
  像把看过的书收进书架，删掉已缓存但不需要显示的工具结果

第三层：Collapse（折叠）
  像把相似的东西归类，把多条相似消息压缩成一条摘要

第四层：Autocompact（自动压缩）
  像请人写读书笔记，用 LLM 生成对话摘要，替换旧消息
```

**类比**：就像你的电脑硬盘满了，先清回收站（Snip），再清缓存（Microcompact），再压缩文件（Collapse），最后实在不行就删旧照片（Autocompact）。

## 模型降级：Plan B 永远存在

```python
try:
    response = await call_claude("claude-opus-4", messages)
except OverloadedError:
    # Opus 太忙了，降级到 Sonnet
    response = await call_claude("claude-sonnet-4", messages)
```

就像你订了头等舱，但飞机满了，自动给你升商务舱。虽然体验差一点，但至少能飞。

---

## 流式执行：不等 Claude 说完就开始干活

传统方式：Claude 说"我要用 3 个工具"，说完才开始执行。

流式方式：Claude 刚说出第一个工具名，就已经开始执行了。

```
传统:  Claude说话(2s) ──── 执行工具A(1s) ── 执行工具B(1s) ── 执行工具C(1s)
       总耗时: 5s

流式:  Claude说话(2s) ──── 执行工具A(1s) ── 执行工具B(1s) ── 执行工具C(1s)
              ↕ 并行          ↕ 并行          ↕ 并行
       总耗时: 3s (省了 2s!)
```

## 安全阀：防止 Claude 陷入死循环

```python
MAX_TURNS = 100  # 最多循环 100 轮

if turn_count > MAX_TURNS:
    return "我尝试了 100 次还是没搞定，可能需要你帮忙"
```

就像你给员工设了截止日期，到时间必须交差，不能无限拖延。

## 面试怎么答

**Q: 为什么用 while True 而不是递归？**
A: 递归有栈深度限制。如果 Claude 调用子 Agent，子 Agent 又调用子 Agent，递归 10 层就可能栈溢出。while True 是扁平循环，没有深度限制。

**Q: 上下文压缩为什么有 4 层？**
A: 每层成本不同。Snip 只是删消息（几乎零成本），Autocompact 要调用 LLM 生成摘要（很贵）。先尝试便宜的，不行再用贵的。
