## 数据流

```
用户输入: "帮我修这个 bug"
    │
    ▼
┌─ Bridge ──────────────────────────────────────────────────────────┐
│  接收消息 → JWT 认证 → 转发给 Query Engine                        │
└──────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─ Query Loop ─────────────────────────────────────────────────────┐
│  ① 压缩旧消息 (上下文管理)                                       │
│  ② 调用 Claude API (streaming)                                  │
│  ③ Claude 返回: "我要用 Grep 搜一下"                             │
│  ④ 权限检查: Grep 是只读的 → 白名单放行                          │
│  ⑤ 执行 Grep → 返回搜索结果                                     │
│  ⑥ Claude 看到结果: "找到了，让我读一下这个文件"                  │
│  ⑦ 执行 Read → 返回文件内容                                      │
│  ⑧ Claude: "我知道怎么修了" → 执行 Edit                          │
│  ⑨ 权限检查: Edit 是写入 → 弹窗问用户 → 用户允许                 │
│  ⑩ 执行 Edit → 修改成功                                         │
│  ⑪ Claude: "让我跑一下测试" → 执行 Bash(pytest)                  │
│  ⑫ 测试通过 → Claude: "修好了！" → 退出循环                     │
└──────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─ Bridge ──────────────────────────────────────────────────────────┐
│  结果 → 同步到 IDE → 高亮修改的文件 → 通知用户                    │
└──────────────────────────────────────────────────────────────────┘
```
