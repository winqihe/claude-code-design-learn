# 05 - 权限系统：AI 的安全带

## 一句话总结

权限系统确保 Claude 不能"偷偷"干危险的事。每次 Claude 想执行一个操作，都要经过三层检查：规则匹配 → 安全分类器 → 用户确认。

---

## 通俗理解

Claude 很聪明，但聪明不代表可以信任。你不会让一个刚认识的聪明人随便操作你的电脑。

Claude Code 的权限系统就像**公司审批流程**：

```
Claude: "老板，我想执行 rm -rf /tmp/test"

第一关：规则匹配（前台小姐姐）
  "rm -rf *" 在黑名单里 → ❌ 拒绝！
  或者: "git status" 在白名单里 → ✅ 放行！

第二关：安全分类器（安检员）
  这个命令看起来有点危险，让 AI 判断一下
  → 安全 → ✅ 放行
  → 危险 → ❌ 拒绝
  → 不确定 → 🤔 交给老板决定

第三关：用户确认（你）
  弹出对话框: "Claude 想执行 xxx，允许吗？"
  [允许一次] [拒绝] [以后都允许]
```

---

## 四种权限模式

```
default (默认模式):
  没有规则匹配的操作 → 问你
  → 日常使用，安全第一

plan (计划模式):
  只允许只读操作，写操作必须问你
  → 规划阶段，防止 Claude 提前动手

bypass (绕过模式):
  跳过所有权限检查
  → CI/CD 自动化，没有人在旁边点确认

auto (自动模式):
  AI 自动判断安全与否
  → 无人值守场景，完全自动化
```

**类比**：
- default = 每笔支出都要你签字
- plan = 只能看账本，不能花钱
- bypass = 给了张无限额信用卡
- auto = AI 财务助理帮你审批小额支出

## 规则匹配：白名单和黑名单

```python
rules = {
    "always_allow": [
        "Bash(git status)",      # git 查看状态，总是允许
        "Bash(git diff)",        # git 查看差异，总是允许
        "Read(*)",               # 读任何文件，总是允许
    ],
    "always_deny": [
        "Bash(rm -rf /)",       # 删除根目录，永远拒绝
        "Bash(sudo *)",         # sudo 命令，永远拒绝
    ],
    "always_ask": [
        "Bash(npm publish)",    # 发布包，总是问你
        "Bash(git push)",       # 推送代码，总是问你
    ],
}
```

规则匹配支持通配符，就像邮箱过滤器一样灵活。

## 安全分类器：AI 审 AI

有些操作不好用规则判断。比如 `Bash(sed -i 's/foo/bar/g' file.txt)`，这是安全的替换还是危险的修改？

这时候就请出**安全分类器**——一个专门判断操作安全性的 AI 模型：

```python
async def classify_safety(tool_name, tool_input):
    # 提取关键信息
    description = f"{tool_name}: {tool_input}"
    # 让分类器判断
    result = await safety_classifier.predict(description)
    return result  # "safe" | "unsafe" | "unsure"
```

```
分类器判断:
  "Bash(ls -la)"        → safe    (只是列目录)
  "Bash(rm -rf /tmp)"   → unsafe  (删东西！)
  "Bash(sed -i ...)"    → unsure  (不确定，问你)
```

## 拒绝追踪：防止 AI "骚扰"用户

如果 Claude 一直请求危险操作，用户会被弹窗烦死。

解决方案：**本地拒绝计数器**

```python
class DenialTracker:
    count: int = 0

    def record_denial(self):
        self.count += 1
        if self.count >= 3:
            # 被拒绝 3 次后，自动切换到"只读模式"
            switch_to_read_only_mode()
```

就像你拒绝了推销员 3 次，他就不会再来了。

## 面试怎么答

**Q: 为什么需要三层权限？**
A: 规则匹配处理明确策略（git 命令总是允许），分类器处理模糊情况（这个 bash 命令安不安全），用户确认是最终兜底。三层互补——规则快但不灵活，分类器灵活但不完美，用户最可靠但最慢。

**Q: bypass 模式为什么存在？**
A: CI/CD 环境中没有人点确认按钮。自动化脚本需要完全跳过权限检查。这是一个必要的"逃生舱口"，但只应该在受控环境中使用。
