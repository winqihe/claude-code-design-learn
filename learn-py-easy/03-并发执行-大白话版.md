# 03 - 并发执行：让 AI 同时干多件事

## 一句话总结

当 Claude 一次要干好几件事时，只读的事情可以同时干（并行），要改东西的事必须一件一件来（串行）。

---

## 通俗理解

想象你是一个项目经理（Claude），手下有一堆任务：

```
任务清单:
  1. 查一下 TODO 注释在哪        ← 只看不动
  2. 找一下所有 .py 文件          ← 只看不动
  3. 读一下 foo.py 的内容         ← 只看不动
  4. 修改 bar.py 的代码           ← 要改东西！
  5. 跑一下测试                   ← 要改东西！（可能产生文件）
```

你会怎么安排？

- 任务 1、2、3 都是**只看不动**的，可以**同时派 3 个人去干**
- 任务 4 要改代码，改完之后任务 5 的测试可能依赖修改结果，所以**必须按顺序来**

这就是**读写分离并发**。

---

## 分区算法：怎么判断哪些能并行？

```python
def partition(tasks):
    batches = []
    current_read_batch = []

    for task in tasks:
        if task.is_read_only:
            # 只读任务：加入当前的"并行批次"
            current_read_batch.append(task)
        else:
            # 写入任务：先把之前的只读批次保存，再单独开一个批次
            if current_read_batch:
                batches.append(("parallel", current_read_batch))
                current_read_batch = []
            batches.append(("serial", [task]))

    if current_read_batch:
        batches.append(("parallel", current_read_batch))

    return batches
```

用上面的例子：

```
输入: [查TODO, 找.py, 读foo.py, 改bar.py, 跑测试]

分区结果:
  Batch 1: [查TODO, 找.py, 读foo.py]  → 并行！同时干
  Batch 2: [改bar.py]                 → 串行，单独干
  Batch 3: [跑测试]                   → 串行，单独干
```

## 为什么要这么设计？

### 只读工具可以并行

```
┌──────────┐  ┌──────────┐  ┌──────────┐
│  Grep    │  │  Glob    │  │  Read    │
│  ~50ms   │  │  ~30ms   │  │  ~20ms   │
└──────────┘  └──────────┘  └──────────┘
     同时执行！总耗时 = max(50, 30, 20) = 50ms
```

因为它们都不改任何东西，谁先谁后无所谓。

### 写入工具必须串行

```
┌──────────┐     ┌──────────┐
│  Edit    │────►│  Bash    │
│  改代码  │     │  跑测试  │
└──────────┘     └──────────┘
     先改代码，再跑测试（测试要看到改后的代码）
```

如果并行执行，测试可能跑的是旧代码，结果就不准了。

## 保守原则：不确定就串行

```python
try:
    parsed = tool.input_schema.validate(input)
    is_safe = tool.is_concurrency_safe(parsed)
except Exception:
    is_safe = False  # 解析失败？默认不安全！
```

就像过安检：不确定这个行李有没有问题？那就开箱检查。宁可慢一点，也不能放过危险品。

## ContextModifier：并行执行的特殊处理

工具执行后可能会修改"环境"（比如 MCP 工具连接后注册了新工具）。

```
串行执行时:
  工具A 执行完 → 立刻应用 A 的环境修改 → 工具B 看到修改后的环境

并行执行时:
  工具A、B、C 同时执行 → 都返回了环境修改 → 怎么办？
  → 先收集所有修改 → 全部执行完后 → 按顺序应用
```

就像几个人同时给你递文件，你不能同时接，得一个一个来。但你可以先让他们都放在桌上，然后按顺序处理。

## 最大并发数：别把系统搞崩了

```python
MAX_CONCURRENCY = 10  # 最多同时执行 10 个工具

semaphore = asyncio.Semaphore(MAX_CONCURRENCY)
```

就像餐厅最多容纳 10 桌客人，满了就得排队。不是因为你不想服务更多客人，而是怕厨房（系统资源）撑不住。

## 面试怎么答

**Q: 为什么写入工具不能并行？**
A: 写入工具可能修改文件系统，后续工具可能依赖前一个工具的修改结果。并行执行会导致竞态条件——就像两个人同时编辑同一个文档，最后谁的内容留下来是不确定的。

**Q: Python 中怎么实现并发限制？**
A: 用 `asyncio.Semaphore(10)`。它像一个计数器，每次执行前减 1，执行完加 1。减到 0 时，新的任务必须等待。
