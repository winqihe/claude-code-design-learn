# Claude Code 的 Harness 哲学与成功之道

## 什么是 Harness？

**Harness**（安全带/挽具）在 AI Agent 领域指的是：**围绕大模型构建的约束、工具和流程，让 AI 能够安全、可靠地完成复杂任务。**

光有一个聪明的模型（Claude）是不够的。就像一匹野马，虽然跑得快，但没有缰绳、马鞍和马蹄铁，你不敢骑。

Claude Code 的 Harness 就是那套"缰绳 + 马鞍 + 马蹄铁"。

---

## Claude Code 为什么这么成功？

### 1. 把 AI 当"不可信的聪明人"

```
❌ 错误的假设: "Claude 很聪明，让它自由发挥就好"
✅ 正确的假设: "Claude 很聪明，但可能犯错，需要层层防护"
```

Claude Code 的每一层设计都体现了这个哲学：

| 设计 | 体现的哲学 |
|------|-----------|
| 工具默认不并发安全 | 不信任工具会正确声明自己 |
| 写入工具必须串行 | 不信任并行执行不会出问题 |
| 三层权限系统 | 不信任 AI 的判断 |
| max_tokens 恢复最多 3 次 | 不信任 AI 不会陷入死循环 |
| /batch 只能用户手动触发 | 不信任 AI 不会滥用大规模操作 |
| Fork 不能偷看中间结果 | 不信任 AI 不会被噪声干扰 |

**核心原则: AI 是能力放大器，不是自主决策者。人类始终在环（Human-in-the-loop）。**

---

### 2. 分层抽象，每层只解决一个问题

```
┌──────────────────────────────────────────────────┐
│  分层架构: 每层独立，职责清晰                      │
│                                                  │
│  工具层:   "能做什么" (读文件、跑命令)            │
│  编排层:   "怎么做" (并行还是串行)                │
│  权限层:   "能不能做" (安全检查)                  │
│  循环层:   "什么时候做" (思考→行动→再思考)        │
│  Agent层: "谁来做" (主Agent还是分身)             │
│  技能层:   "按什么套路做" (预定义工作流)          │
│  MCP层:   "还能连接什么" (外部工具扩展)           │
│                                                  │
│  每层可以独立修改、测试、替换                      │
│  修改权限系统不影响工具系统                        │
│  添加新工具不需要改循环逻辑                        │
└──────────────────────────────────────────────────┘
```

**类比**：就像乐高积木。每个积木块（层）有标准的接口（凸起和凹槽），可以自由组合。你想加个新工具？插上去就行。想换种权限策略？拔下来换一个。

---

### 3. 渐进式信任（Progressive Trust）

Claude Code 不是一开始就给 AI 全部权限，而是**逐步建立信任**：

```
第一次使用:
  → 每个操作都要用户确认 (default 模式)

使用一段时间后:
  → 用户把常用操作加入白名单 (git status, Read 等)
  → 这些操作不再需要确认

完全信任后:
  → 切换到 auto 模式 (AI 自动判断)
  → 或 bypass 模式 (CI/CD 环境)

/batch 技能:
  → 即使在 auto 模式下，/batch 也只能用户手动触发
  → 大规模操作永远需要人类审批
```

**类比**：就像新员工入职。第一个月什么都让你审批，三个月后小额支出不用审批，一年后给你部门预算权限。但公司并购这种大事，永远需要董事会批准。

---

### 4. 让 AI 自己管理复杂度

Claude Code 最精妙的设计之一：**让 AI 自己决定用哪个工具、什么时候用、怎么用。**

```python
# ❌ 传统方式: 硬编码工作流
if task == "fix_bug":
    grep(error_message)
    read(found_file)
    edit(fix)
    bash(test)

# ✅ Claude Code 方式: AI 自主决策
while True:
    response = await claude(messages)
    # Claude 自己决定下一步做什么
    # 可能搜代码、读文件、改代码、跑测试...
    # 顺序和次数都由 AI 决定
```

这不是"放任 AI 乱来"，而是**在安全约束内给 AI 最大自由度**。权限系统是护栏，工具系统是方向盘，循环系统是引擎。

---

### 5. 成本意识贯穿始终

Claude Code 对 token 成本极其敏感：

```
启动优化:
  ├── 并行预读 Keychain (省 65ms)
  └── 并行预读 MDM 配置 (省 65ms)

上下文管理:
  ├── 4 层压缩 (省 token)
  ├── 延迟加载工具 (省 prompt token)
  └── Agent 列表从工具描述移到消息 (省 10% cache token)

Fork Agent:
  ├── 共享 prompt cache (省大量 token)
  └── 不偷看中间结果 (避免噪声 token)

技能发现:
  ├── 97% 查找无结果 → 异步预取 (省时间)
  └── find_write_pivot 守卫 (非写操作提前返回)
```

**类比**：就像一个精打细算的管家。不是省钱到抠门，而是每一分钱都花在刀刃上。

---

### 6. 优雅降级（Graceful Degradation）

当出问题时，Claude Code 不会崩溃，而是优雅降级：

```
模型过载 → 降级到 Sonnet
上下文过长 → 4 层压缩
工具执行失败 → 返回错误信息，继续循环
MCP 连接断开 → 标记工具不可用
技能文件提取失败 → 不影响技能运行
权限检查异常 → 默认拒绝（安全第一）
```

**类比**：就像飞机。一个引擎坏了，还能用另一个飞。液压系统故障，还有手动备份。永远有 Plan B。

## Harness 的核心公式

```
成功的 AI Agent = 聪明的模型 × 精密的 Harness

其中:
  聪明的模型 = Claude (Anthropic 提供)
  精密的 Harness = Claude Code (52 万行代码)

Harness 的组成部分:
  ├── 工具系统 (手和眼睛)
  ├── 运行循环 (心跳)
  ├── 并发执行 (效率)
  ├── 多 Agent (分身术)
  ├── 权限系统 (安全带)
  ├── 技能系统 (武功秘籍)
  ├── MCP 协议 (万能插头)
  └── 上下文管理 (记忆宫殿)
```

## 为什么其他 Agent 框架做不到？

| 维度 | Claude Code | 其他框架 |
|------|-------------|---------|
| **工具抽象** | 统一接口 + 安全默认值 | 各自为政，没有统一标准 |
| **权限系统** | 三层防线 + 渐进信任 | 要么没有，要么太简单 |
| **并发执行** | 读写分离 + 信号量 | 大多全串行 |
| **多 Agent** | Fork + Worktree + 通知 | 大多是简单的函数调用 |
| **上下文管理** | 4 层压缩管道 | 大多只有简单的截断 |
| **成本优化** | 贯穿每个环节 | 大多不关心 |
| **优雅降级** | 每个环节都有 fallback | 大多出错就崩 |
| **流式执行** | LLM 和工具并行 | 大多等 LLM 说完再执行 |

## 给 Agent 开发者的 7 条黄金法则

```
1. 不信任原则
   AI 很聪明，但可能犯错。每个决策都要有安全检查。

2. 分层原则
   每层只解决一个问题。工具层不管权限，权限层不管执行。

3. 渐进信任原则
   从最严格的权限开始，逐步放宽。永远保留人类审批的选项。

4. 成本意识原则
   Token 是钱。能省则省，但不能省到影响效果。

5. 优雅降级原则
   任何环节都可能失败。每个失败都要有 fallback。

6. 人类在环原则
   AI 是工具，不是决策者。关键操作必须人类确认。

7. 简单优先原则
   能用简单方案就不要用复杂方案。Claude Code 用 while True 而不是复杂的状态机。
```

---

## 总结

Claude Code 的成功不是因为 Claude 模型本身（虽然它确实很强），而是因为围绕它构建的 **Harness**：

- **安全** — 三层权限 + 渐进信任 + 人类在环
- **高效** — 并发执行 + 流式工具 + 成本优化
- **可靠** — 优雅降级 + 安全阀 + 4 层压缩
- **可扩展** — MCP 协议 + 技能系统 + 工厂函数
- **简洁** — 分层抽象 + 统一接口 + 简单优先

**一个聪明的模型 + 一个精密的 Harness = 一个强大的 Agent。**

这就是 Claude Code 成功的秘密。
